## 📌 프로젝트 요약 (Project Overview)
이번 프로젝트에서는 "컴퓨터가 이미지 속 물체의 위치를 어떻게 찾아내는 걸까?"라는 원초적인 궁금증을 바탕으로 단순히 이미 만들어진 라이브러리를 가져다 쓰는 것에 그치지 않고, 그 내부에서 돌아가는 핵심 로직들을 하나씩 직접 구현하며 객체 탐지의 '기초 체력'을 기르는 것을 목표로 삼았습니다. 바운딩 박스를 계산하는 아주 기초적인 수학부터 시작해서, 여러 개의 박스 중 가장 정확한 것만 남기는 알고리즘, 그리고 실제로 물체를 찾아내는 신경망을 설계하는 과정까지를 차례대로 담고 있습니다.

## 🎯 핵심 목표 (Motivation)
| 핵심 질문 (Question) | 해결하기 위한 노력 (Approach) | 깨달은 점 (Insight) &emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp; |
| :--- | :--- | :--- |
| **"박스가 겹친 정도를 어떻게 수치화할까?"** | IoU(Intersection over Union) 계산식을 직접 함수로 구현 | 컴퓨터가 이미지 좌표를 인식하고 면적을 계산하는 기초 기하학 원리 이해 |
| **"지저분하게 겹친 박스들을 어떻게 치울까?"** | NMS(Non-Maximum Suppression) 알고리즘 로직 설계 | 수많은 예측 중 가장 높은 확률만 남기는 효율적인 후처리 데이터 필터링 방식 체득 |
| **"하나의 신경망이 두 가지 일을 할 수 있을까?"** | 위치(Regression)와 분류(Classification)를 동시에 수행하는 <br>모델 구조 설계 | 서로 다른 성격의 정답(좌표와 이름)을 함께 학습시키는 멀티태스크 학습의 기초 확립 |

## 🛠️ 핵심 구현 기술 (Technical Implementation)

| 핵심 개념 (Core Concept) | 내 손으로 구현한 방식 (Implementation Details) | 직접 부딪히며 깨달은 점 (Key Insight) |
| :--- | :--- | :--- |
| **IoU 계산 알고리즘** | 눈으로만 보던 교집합/합집합 계산을 `calculate_iou` 함수로 직접 짜봤습니다. 하다 보니 분모가 0이 돼서 프로그램이 뻗는 걸 겪고, 아주 작은 값(`1e-16`)을 더해주는 방어 코드를 추가했습니다. | 그냥 수학 공식인 줄 알았는데, 컴퓨터가 픽셀 좌표를 가지고 면적을 계산하는 기하학적 원리를 바닥부터 제대로 이해하게 되었습니다. |
| **NMS 후처리 로직** | 모델이 똑같은 물체에 박스를 여러 개 그리는 문제를 해결하려고, 제일 점수가 높은 '대장' 박스만 남기고 겹치는 건 싹 지워버리는 청소기 같은 `nms` 함수를 만들었습니다. | 화면에 덕지덕지 붙어있던 지저분한 박스들이 이 로직 하나로 깔끔하게 정리되는 걸 보면서, 데이터 후처리의 엄청난 힘을 체감했습니다. |
| **Multi-task 신경망** | ResNet18을 가져와서 '위치'를 찾는 길과 '이름'을 맞추는 길 두 갈래로 나눴습니다. 두 미션의 난이도가 달라서 `alpha`, `beta` 값으로 학습 밸런스를 조절할 수 있게 만들었습니다. | 오차(Loss) 두 개를 생각 없이 그냥 더해버리면 모델이 한쪽 미션만 편식한다는 걸 깨달았습니다. 균형을 잡아주는 유연한 설계가 필수더라고요. |
| **학습 안정화 세팅** | 처음엔 모델이 정답을 전혀 못 찾아서 헤맸는데, 커스텀 레이어에 초기화(`He Init`)를 제대로 해주고 `BatchNorm1d`를 끼워 넣으니 그제야 학습이 정상적으로 돌아가기 시작했습니다. | 이론으로만 달달 외우던 최적화 기법들이, 실제로는 딥러닝 모델의 숨통을 틔워주는 생명줄이라는 걸 몸소 느꼈습니다. |

## 📊 학습 성과 및 지표 (Results)

| 항목 (Metrics) | 학습 초기 (Epoch 1) | 학습 완료 (Epoch 100) | 결과에 대한 해석 (Interpretation) |
| :--- | :---: | :---: | :--- |
| **위치 오차 (BBox MSE Loss)** | 360.25 | **1.24** | 초기에는 박스가 엉뚱한 곳에 그려졌으나, 학습 후 실제 물체 크기에 딱 맞게 조절됨 |
| **분류 오차 (Class CE Loss)** | 1.65 | **0.02** | 초반엔 물체 이름을 헷갈려 했지만, 나중에는 거의 100%에 가까운 확률로 이름을 맞춤 |
| **전체 오차 (Total Loss)** | 361.90 | **1.26** | 두 가지 목표(위치+이름)가 서로 충돌하지 않고 안정적으로 줄어들며 학습이 수렴 |

**100번의 반복 학습(Epochs)을 거치며 모델이 어떻게 정답을 찾아가는지 관찰했습니다. 초기에는 박스 위치를 전혀 잡지 못했지만, 학습이 진행될수록 위치와 종류를 모두 정확하게 맞추기 시작했습니다.**

### 💡 주요 성과 (Key Achievement)

- **손실 함수(Loss) 99% 이상 감소**: 초기 오차에 비해 최종 오차를 획기적으로 줄이며 모델의 예측 정밀도를 확보했습니다.
- **안정적인 수렴**: 학습 정체 구간이 있었으나, 배치 정규화(Batch Normalization)를 적용하여 흔들림 없이 학습을 마무리할 수 있었습니다.

## 💡 회고록 (Retrospective)

&emsp;&emsp;이번 프로젝트를 통해 AI 모델 개발에서 "이론적 기반"이 실무 엔지니어링에 얼마나 큰 영향을 미치는지 재확인했습니다. 직접 IoU 수식을 짜보고, NMS 로직을 설계하며, 두 갈래의 신경망을 연결해 보는 과정을 통해 **"당연하게 돌아가는 것은 없다"** 는 소중한 교훈을 얻었습니다. 픽셀 좌표 하나하나를 계산하며 분모가 0이 되어 프로그램이 멈추는 작은 에러를 잡았을 때, 비로소 '엔지니어링의 디테일'이 무엇인지 조금이나마 깨달을 수 있었습니다. 또한 위치와 종류라는 서로 다른 두 가지 정답을 동시에 맞추기 위해 신경망의 균형을 잡는 과정은 딥러닝 아키텍처의 기본을 느끼게 해 준 값진 경험이었습니다. 특히, 단순히 모델을 돌려보는 것에 그치지 않고 지저분한 연습장 같던 코드를 파일별로 모듈화하여 정리하며, '돌아가는 코드'를 넘어 '관리할 수 있는 코드'의 가치도 배울 수 있었습니다. 처음으로 내가 짠 로직을 거친 빨간 박스가 이미지 속 물체를 정확히 찾아냈을 때의 소중한 경험은, 앞으로 제가 AI 엔지니어로 성장해 나가는 데 있어 가장 강력한 동기부여가 될 것입니다. 
<br>&emsp;&emsp;여기서 멈추지 않고, 이제는 '기초'를 넘어 '실전'으로 나아가려 합니다. 본 프로젝트에서 구현한 로직들이 실제 SOTA(State-of-the-Art) 모델인 YOLOv5에서는 어떻게 최적화되어 작동하는지, 그리고 실시간성(Real-time)이라는 실무적 제약을 어떻게 극복하는지 탐구하는 것이 저의 다음 목표입니다. 이어지는 프로젝트 yolov5-pothole-detector에서는 이번에 배운 이론적 토대 위에 '실무 서비스 적용'과 '성능 최적화'라는 경험을 쌓아, 원리와 실천을 겸비한 엔지니어로 성장해 나가겠습니다.

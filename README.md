## 📌 프로젝트 요약 (Project Overview)
이번 프로젝트에서는 "컴퓨터가 이미지 속 물체의 위치를 어떻게 찾아내는 걸까?"라는 원초적인 궁금증을 바탕으로 단순히 이미 만들어진 라이브러리를 가져다 쓰는 것에 그치지 않고, 그 내부에서 돌아가는 핵심 로직들을 하나씩 직접 구현하며 객체 탐지의 '기초 체력'을 기르는 것을 목표로 삼았습니다. 바운딩 박스를 계산하는 아주 기초적인 수학부터 시작해서, 여러 개의 박스 중 가장 정확한 것만 남기는 알고리즘, 그리고 실제로 물체를 찾아내는 신경망을 설계하는 과정까지를 차례대로 담고 있습니다.

## 🛠️ 핵심 구현 기술 (Technical Implementation)
프로젝트의 핵심은 단순히 모델을 돌려보는 것이 아니라, 객체 탐지의 '기초 로직'을 직접 코딩하며 원리를 깨닫는 것이었습니다. 제가 가장 공들여 구현한 세 가지 기술적 포인트를 정리했습니다.

### 1. IoU(Intersection over Union): 정답과 얼마나 가까운가?
모델이 예측한 박스가 정답과 얼마나 유사한지 판단하는 기준을 직접 만들었습니다.
- **핵심 로직**: 두 박스가 겹치는 면적(Intersection)을 전체 합친 면적(Union)으로 나누어 0~1 사이의 점수를 냅니다.
- **구현 디테일**: 분모가 0이 되어 프로그램이 멈추는 것을 막기 위해 아주 작은 값(`1e-16`)을 더해주는 예외 처리를 추가했습니다. 덕분에 어떤 상황에서도 에러 없이 안정적으로 점수를 계산합니다.



### 2. NMS(Non-Maximum Suppression): 지저분한 박스들 정리하기
모델은 하나의 물체에 대해 수많은 박스를 그리는 경향이 있습니다. 이를 하나로 합치는 '청소부' 로직을 구현했습니다.
- **해결 방식**: 
  1. 먼저 자신감(Score)이 가장 높은 박스를 '대장'으로 뽑습니다.
  2. 그 다음, 대장과 너무 많이 겹치는(IoU가 높은) 나머지 박스들은 중복으로 간주해 제거합니다.
- **배운 점**: 이 과정을 거치면 화면에 덕지덕지 붙어있던 박스들이 사라지고, 가장 정확한 박스 하나만 남게 됩니다.



### 3. 멀티태스크 신경망: "어디에 있고, 무엇인가?"를 동시에 맞추기
물체의 위치를 찾는 일(Regression)과 이름을 맞추는 일(Classification)을 한 번에 수행하는 구조를 설계했습니다.
- **Backbone**: 이미 검증된 ResNet18 모델을 가져와 이미지의 특징을 잘 뽑아내도록 활용했습니다.
- **Double Heads**: 특징을 뽑은 뒤 길을 두 갈래로 나누었습니다.
  - **좌표 예측**: 박스의 위치 $(x, y, w, h)$를 0~1 사이 값으로 출력합니다.
  - **종류 예측**: 이 물체가 무엇인지 확률로 나타냅니다.
- **최적화**: 두 가지 다른 성격의 정답을 동시에 학습시키기 위해, 두 개의 Loss(MSE, CrossEntropy)를 합쳐서 모델이 균형 있게 배울 수 있도록 유도했습니다.

## 🔍 트러블슈팅 (Troubleshooting)

| 문제 상황 | 분석 및 해결 방안 | 성과 |
| :--- | :--- | :--- |
| **좌표 발산 현상** | **Sigmoid 정규화 적용**: Regression Head의 마지막 출력단에 Sigmoid를 배치하여 좌표 범위를 $0 \sim 1$로 강제 제한 | 학습 수렴 안정성 확보 |
| **학습 정체 구간** | **Batch Normalization 추가**: Linear 레이어 직후 BatchNorm1d를 배치하여 내부 공변량 변화를 제어 | 최종 Loss 15% 추가 감소 |
| **데이터 타입 충돌** | **Type-Agnostic 연산 설계**: `ops.py` 내부에서 입력 타입을 체크하고 자동으로 변환하는 로직 구현 | 장치 독립적 추론 환경 구축 |

## 💡 회고록 (Retrospective)

&emsp;&emsp;이번 프로젝트를 통해 AI 모델 개발에서 **"이론적 기반"**이 실무 엔지니어링에 얼마나 큰 영향을 미치는지 재확인했습니다. 단순히 라이브러리의 함수를 호출할 때는 알 수 없었던 NMS의 필터링 원리와 Multi-task Loss의 가중치 밸런싱 과정을 직접 코딩하며, 탐지 모델의 내부 동작을 완벽히 내재화할 수 있었습니다. 특히, 학습 초기 모델이 바운딩 박스를 엉뚱한 곳에 그리던 문제를 Sigmoid와 BatchNorm으로 해결하며 딥러닝 아키텍처 최적화의 핵심을 체감했습니다.
